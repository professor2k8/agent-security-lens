# Contributing

Contributions should improve the public MCP interface, schemas, local fallback rules, ecosystem profiles, tests, or documentation.

Do not submit private component data, credentials, proprietary source code, unverified accusations, or internal service details.

## Pull Requests

1. Keep changes focused.
2. Add or update tests for behavior changes.
3. Run `npm run verify:public`.
4. Describe the observable behavior and evidence.
5. Do not describe a component as malicious without reproducible evidence.

Security vulnerabilities must be reported privately as described in [SECURITY.md](SECURITY.md).
